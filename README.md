## 📖 为什么做这个项目


在学习和求职这两件事上，**结构化的信息处理**本身就是一种隐性成本——整理笔记、筛选岗位、分析匹配度……这些流程完全可以让 AI 自动完成，让你把精力真正放在理解、准备和行动上。

于是我搭建了：
- 📝 **Agent Skills**：通过agent skills 能更大地助力学习效率和求职效率，将讲义一键转化为结构化学习文件，将简历+JD一键转化为投递优先级分析

---

## ✨ 特点

- 🍱 **开箱即用**：上传PPT/PDF，说一句话，直接得到完整HTML批注文件
- 🎨 **复制即用**：无需重复造轮子，精心设计的双栏排版和配色
- 🔄 **持续更新**：不断收录新的学习/求职场景 skill

> 不要在整理笔记上浪费时间，把精力留给真正重要的事。

## Part I：Agent Skills

> 🎯 **适用对象**：本部分内容主要面向经常使用 Claude、Claude Code 等 AI 工具的用户
>
> 💡 **使用说明**：Agent Skills 是一种可被 AI 助手（如 Claude）加载的扩展能力包，内含针对特定任务的流程、规范与模板。配置对应 Skill 后，在对话中直接描述需求，即可触发应流程，无需记忆复杂 prompt。

### Skills 的配置

推荐通过 **OpenSkills** 安装，支持直接从 GitHub 仓库一键加载。

**前置依赖：**
- Node.js 20.6+（含 npm）
- Git

**安装 OpenSkills：**

```bash
npx openskills --version
# 或全局安装
npm i -g openskills
```

**一键安装本仓库全部 Skills：**

```bash
npx openskills install Daisssy40/claude-skills
```

执行后 OpenSkills 会弹出交互式选择，勾选需要的 Skill 即可，默认全部安装。

## 📖 study-annotator

> 将PPT讲义或课程材料，转化为一个完整的**双栏学习批注HTML文件**，整合康奈尔笔记、间隔复习标记和思维导图结构，帮助高效学习和备考。

### 功能亮点

| 功能 | 说明 |
|------|------|
| 🎨 五色高亮系统 | 核心概念 / 考点 / 数据 / 操作步骤 / 方法论，一眼区分 |
| 📋 双栏批注 | 左栏原文精简版，右栏中文批注，严格对齐 |
| 🗝 康奈尔要点 | 每段提炼关键词、逻辑重心、章节关联 |
| ⚠️ 记忆难点标注 | 标出易错点，附记忆技巧，Hard/Medium/Easy 分级 |
| 🧪 自测题（折叠） | 每段配1道自测题，答案默认折叠，点击展开 |
| 🧠 章节知识树 | 每章末尾自动生成思维导图骨架（3层层级） |
| 🔁 间隔复习提示 | 标出本章最需要下次重点检验的2-3个概念 |
| ✅ 全文速查清单 | 底部汇总所有高频考点，可勾选形式 |
| 📌 概念索引 | 中英对照，附1句话定义 |

### 触发方式

上传PPT/PDF后，直接说：

```
帮我整理这个PPT，做成学习批注文件
```

```
结合这份讲义，生成双栏学习批注HTML，我需要备考
```

```
把这个课件做成康奈尔笔记格式的HTML
```

### 文件结构

```
study-annotator/
├── SKILL.md                        ← 核心工作流与规范
└── references/
    └── html-template.md            ← 完整HTML/CSS模板
```

## 💼 job-application-filter

> 将你的简历与多个JD进行系统性匹配分析，输出**投递优先级排序**、每岗匹配度评分、风险点和策略建议，适用于实习、校招、社招全场景。

### 功能亮点

| 功能 | 说明 |
|------|------|
| 🚫 硬性过滤 | 自动排除届次/实习时长/城市等硬性不满足的岗位 |
| 📊 五维匹配评分 | 关键词命中率 / 面试竞争力 / 故事线完整度 / 申请要求 / 岗位稀缺性 |
| 🏆 岗位价值分级 | S/A/B/C 级分类（ByteIntern > 大厂日常 > 中型公司） |
| 🎯 内推码策略 | 有内推码时自动切换机筛→人工筛逻辑，重新排序 |
| 📋 投递志愿建议 | 针对"最多N个/第一志愿优先"等限制给出具体顺序 |
| ⚠️ 逐岗风险提示 | 标出弱项、包装建议、额外申请材料提醒 |

### 触发方式

上传简历 + 多个JD后，直接说：

```
帮我看看这几个岗位，我应该投哪个
```

```
帮我排一下投递优先级，我有内推码
```

```
这些岗位哪个最匹配我，先投哪个
```

### 输出示例

```
| 优先级 | 岗位名称 | 公司 | 级别 | 匹配分 | 核心理由   | 主要风险   |
|--------|---------|------|------|--------|-----------|-----------|
| 🥇 冲  | 产品实习 | 字节 | S级  |  9/10  | 经历高度匹配 | 竞争激烈  |
| 🥈 稳  | 运营实习 | 美团 | A级  |  8/10  | 技能命中率高 | 需cover letter |
| 🥉 补  | 数据实习 | 中型 | B级  |  7/10  | 保底背书   | 无明显亮点 |
```

### 文件结构

```
job-application-filter/
└── SKILL.md                        ← 完整筛选逻辑与评分规范
```
